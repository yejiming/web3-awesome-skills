# BingX Copy Trade USDT-M Perpetual Contracts — API Reference

**Base URLs:** see [`references/base-urls.md`](../references/base-urls.md) | **Auth:** HMAC-SHA256 — see [`references/authentication.md`](../references/authentication.md) | **Response:** `{ "code": 0, "msg": "", "data": ... }`

**Common parameters** (apply to all endpoints below):

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| timestamp | int64 | Yes | Request timestamp in milliseconds |
| recvWindow | int64 | No | Request validity window in ms, max 5000 |

> **Note:** Endpoint paths use the `/openApi/copyTrading/v1/` prefix per the official BingX API documentation. Order management APIs use `/swap/trace/` path, while trader profile and profit APIs use `/PFutures/` path.

---

## 1. Trader's Current Orders

### Query Trader's Current Open Orders

`GET /openApi/copyTrading/v1/swap/trace/currentTrack`

Query the trader's currently open (active) copy trade USDT-M perpetual orders.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `symbol` | string | Yes | Filter by trading pair, e.g. `BTC-USDT` |
| `offset` | int | No | Offset, default is 0. |
| `limit` | int | No | Number of results |

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `total` | int64 | Total number of current open orders |
| `list` | array | Array of order objects (see below) |

**Order Object:**

| Field | Type | Description |
|-------|------|-------------|
| `orderId` | string | Copy trade order ID |
| `symbol` | string | Trading pair, e.g. `BTC-USDT` |
| `side` | string | Order direction: `BUY` or `SELL` |
| `positionSide` | string | Position side: `LONG`, `SHORT`, or `BOTH` |
| `type` | string | Order type: `MARKET` or `LIMIT` |
| `origQty` | string | Original order quantity |
| `executedQty` | string | Filled quantity |
| `price` | string | Order price (for LIMIT orders; `"0"` for MARKET) |
| `avgPrice` | string | Average fill price |
| `leverage` | int64 | Leverage used for the order |
| `unrealizedProfit` | string | Current unrealized PnL (USDT) |
| `marginType` | string | Margin type: `ISOLATED` or `CROSSED` |
| `createTime` | int64 | Order creation timestamp in milliseconds |
| `updateTime` | int64 | Last update timestamp in milliseconds |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "total": 2,
    "list": [
      {
        "orderId": "1736011869418901234",
        "symbol": "BTC-USDT",
        "side": "BUY",
        "positionSide": "LONG",
        "type": "MARKET",
        "origQty": "0.01",
        "executedQty": "0.01",
        "price": "0",
        "avgPrice": "65000.00",
        "leverage": 10,
        "unrealizedProfit": "12.50",
        "marginType": "ISOLATED",
        "createTime": 1704067200000,
        "updateTime": 1704067201000
      }
    ]
  }
}
```

## 2. Close Position by Order Number

### Traders Close Positions According to the Order Number

`POST /openApi/copyTrading/v1/swap/trace/closeTrackOrder`

Close (liquidate) a copy trade USDT-M perpetual position identified by its order number. This triggers a market close of the position associated with the given copy trade order.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `positionId` | int64 | Yes | Order number with order |

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `orderId` | string | The close order ID generated by the system |
| `symbol` | string | Trading pair, e.g. `BTC-USDT` |
| `side` | string | Close order direction (opposite of the open side) |
| `type` | string | Order type — `MARKET` |
| `origQty` | string | Original quantity to close |
| `executedQty` | string | Filled quantity |
| `avgPrice` | string | Average fill price |
| `realizedProfit` | string | Realized PnL from closing this position (USDT) |
| `status` | string | Order status: `NEW`, `PARTIALLY_FILLED`, `FILLED`, `CANCELED` |
| `createTime` | int64 | Order creation timestamp in milliseconds |
| `updateTime` | int64 | Last update timestamp in milliseconds |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "orderId": "1736011869418909876",
    "symbol": "BTC-USDT",
    "side": "SELL",
    "type": "MARKET",
    "origQty": "0.01",
    "executedQty": "0.01",
    "avgPrice": "65500.00",
    "realizedProfit": "5.00",
    "status": "FILLED",
    "createTime": 1704070800000,
    "updateTime": 1704070801000
  }
}
```

## 3. Set Take Profit and Stop Loss by Order Number

### Traders Set Take Profit and Stop Loss Based on Order Numbers

`POST /openApi/copyTrading/v1/swap/trace/setTPSL`

Set or update the take profit (TP) and/or stop loss (SL) prices for an active copy trade USDT-M perpetual order. At least one of `takeProfit` or `stopLoss` must be provided.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `positionId` | int64 | Yes | Order number with order |
| `takeProfitMarkPrice` | float64 | Yes | Set the price of the stop profit and stop loss mark, and the price of the stop profit and stop loss mark must be one of |
| `stopLossMarkPrice` | float64 | Yes | Stop loss mark price |

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `orderId` | string | The copy trade order ID |
| `symbol` | string | Trading pair, e.g. `BTC-USDT` |
| `takeProfit` | string | Confirmed take profit price |
| `stopLoss` | string | Confirmed stop loss price |
| `updateTime` | int64 | Timestamp when TP/SL was applied in milliseconds |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "orderId": "1736011869418901234",
    "symbol": "BTC-USDT",
    "takeProfit": "70000.00",
    "stopLoss": "60000.00",
    "updateTime": 1704068000000
  }
}
```

## 4. Personal Trading Overview

### Personal Trading Overview

`GET /openApi/copyTrading/v1/PFutures/traderDetail`

Query the current trader's personal copy trade USDT-M perpetual performance overview, including total profit, win rate, follower count, and assets under management (AUM).

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `daySize` | int | No | Profit rate over the last N days, N must be 7, 30, 90, or 180 |

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `totalProfit` | string | Total realized profit (USDT) across all closed copy trade positions |
| `totalOrders` | int64 | Total number of copy trade orders placed |
| `winOrders` | int64 | Number of profitable (winning) orders |
| `lossOrders` | int64 | Number of losing orders |
| `winRate` | string | Win rate as a decimal, e.g. `"0.65"` = 65% |
| `totalFollowers` | int64 | Number of followers currently copying this trader |
| `totalCopyAmount` | string | Total amount under management by followers (USDT) |
| `avgProfitRate` | string | Average profit rate per trade as a decimal |
| `totalProfitRate` | string | Total profit rate since the trader started as a decimal |
| `createTime` | int64 | Trader account creation timestamp in milliseconds |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "totalProfit": "3456.78",
    "totalOrders": 250,
    "winOrders": 165,
    "lossOrders": 85,
    "winRate": "0.66",
    "totalFollowers": 120,
    "totalCopyAmount": "250000.00",
    "avgProfitRate": "0.028",
    "totalProfitRate": "0.346",
    "createTime": 1680000000000
  }
}
```

## 5. Profit Overview

### Profit Overview

`GET /openApi/copyTrading/v1/PFutures/profitHistorySummarys`

Query aggregated profit summary for the copy trade USDT-M perpetual account over a specified time range.

**Parameters:**

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `totalProfit` | string | Total realized profit (USDT) in the queried period |
| `totalLoss` | string | Total realized loss (USDT) in the queried period |
| `netProfit` | string | Net profit = totalProfit − totalLoss |
| `profitRate` | string | Net profit rate as a decimal |
| `totalOrders` | int64 | Total closed orders in the queried period |
| `winOrders` | int64 | Number of profitable orders in the queried period |
| `winRate` | string | Win rate as a decimal |
| `startTime` | int64 | Actual start of the queried period (milliseconds) |
| `endTime` | int64 | Actual end of the queried period (milliseconds) |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "totalProfit": "1200.50",
    "totalLoss": "320.80",
    "netProfit": "879.70",
    "profitRate": "0.042",
    "totalOrders": 60,
    "winOrders": 40,
    "winRate": "0.667",
    "startTime": 1704067200000,
    "endTime": 1706745600000
  }
}
```

## 6. Profit Details

### Profit Details (Paginated)

`GET /openApi/copyTrading/v1/PFutures/profitDetail`

Query per-order profit details for copy trade USDT-M perpetual orders, paginated. Returns a profit/loss breakdown for each individual closed copy trade order.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `startTime` | int64 | No | Start timestamp in milliseconds |
| `endTime` | int64 | No | End timestamp in milliseconds |
| `pageIndex` | int64 | Yes | Page number, must be greater than 0 (default: 1) |
| `pageSize` | int64 | Yes | Records per page, maximum 100 (default: 20) |

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `total` | int64 | Total number of records |
| `list` | array | Array of profit detail objects (see below) |

**Profit Detail Object:**

| Field | Type | Description |
|-------|------|-------------|
| `orderId` | string | Copy trade order ID |
| `symbol` | string | Trading pair, e.g. `BTC-USDT` |
| `side` | string | Original order direction: `BUY` or `SELL` |
| `positionSide` | string | Position side: `LONG`, `SHORT`, or `BOTH` |
| `openPrice` | string | Average entry (open) price |
| `closePrice` | string | Average exit (close) price |
| `qty` | string | Position quantity |
| `leverage` | int64 | Leverage used |
| `profit` | string | Realized profit (USDT); negative value means a loss |
| `profitRate` | string | Profit rate as a decimal |
| `commission` | string | Trading commission charged (USDT) |
| `openTime` | int64 | Position open timestamp in milliseconds |
| `closeTime` | int64 | Position close timestamp in milliseconds |

## 7. Set Commission Rate

### Set Commission Rate

`POST /openApi/copyTrading/v1/PFutures/setCommission`

Set the commission rate that followers pay to this trader for copy trading USDT-M perpetual contracts.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `newCommission` | string | Yes | The commission rate to set, must be greater than 10% and less than the current level. Silver level: 16%, Gold level: 20%|

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `commissionRate` | string | The confirmed new commission rate |
| `updateTime` | int64 | Timestamp when the commission rate was updated in milliseconds |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "commissionRate": "0.001",
    "updateTime": 1704068000000
  }
}
```

## 8. Trader Gets Copy Trading Pairs

### Trader Gets Copy Trading Pairs

`GET /openApi/copyTrading/v1/PFutures/tradingPairs`

Retrieve the list of USDT-M perpetual trading pairs that are available or configured for this trader's copy trading.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `contractType` | string | Yes | Contract type, SFUTURES: Standard contract, PFUTURES: USDT-margined contract |

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `list` | array | Array of copy trading pair objects (see below) |

**Copy Trading Pair Object:**

| Field | Type | Description |
|-------|------|-------------|
| `symbol` | string | Trading pair, e.g. `BTC-USDT` |
| `status` | string | Pair status: `TRADING` or `BREAK` |
| `baseAsset` | string | Base asset name, e.g. `BTC` |
| `quoteAsset` | string | Quote asset name, e.g. `USDT` |
| `pricePrecision` | int64 | Number of decimal places for price |
| `quantityPrecision` | int64 | Number of decimal places for quantity |
| `maxLeverage` | int64 | Maximum leverage allowed for this pair |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "list": [
      {
        "symbol": "BTC-USDT",
        "status": "TRADING",
        "baseAsset": "BTC",
        "quoteAsset": "USDT",
        "pricePrecision": 1,
        "quantityPrecision": 3,
        "maxLeverage": 125
      },
      {
        "symbol": "ETH-USDT",
        "status": "TRADING",
        "baseAsset": "ETH",
        "quoteAsset": "USDT",
        "pricePrecision": 2,
        "quantityPrecision": 3,
        "maxLeverage": 100
      }
```

## Error Codes

Standard BingX error codes apply. Common errors for Copy Trade endpoints:

| Code | Description |
|------|-------------|
| `0` | Success |
| `-1100` | Illegal characters in a parameter |
| `-1102` | Mandatory parameter was not sent or is empty |
| `-1121` | Invalid symbol |
| `-2013` | Order does not exist |
| `-2014` | API key format invalid |
| `-2015` | Invalid API key, IP, or permissions |
| `80001` | Copy trade order not found |
| `80002` | Order already closed or not eligible for this operation |
| `80003` | TP/SL price must be greater than zero |
| `80004` | Commission rate out of allowed range |
