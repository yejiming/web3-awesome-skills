# BingX Copy Trade Spot — API Reference

**Base URLs:** see [`references/base-urls.md`](../references/base-urls.md) | **Auth:** HMAC-SHA256 — see [`references/authentication.md`](../references/authentication.md) | **Response:** `{ "code": 0, "msg": "", "data": ... }`

**Common parameters** (apply to all endpoints below):

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| timestamp | int64 | Yes | Request timestamp in milliseconds |
| recvWindow | int64 | No | Request validity window in ms, max 5000 |

> **Note:** Copy Trade Spot endpoints are documented at `https://bingx-api.github.io/docs-v3/#/en/Copy%20Trade/Spot%20Trading/`. The paths below use the `/openApi/copyTrading/v1/` prefix per the official BingX API documentation.

---

## 1. Trader Sells Spot Assets

### Trader Sells Spot Assets Based on Buy Order Number

`POST /openApi/copyTrading/v1/spot/trader/sellOrder`

Sell the spot assets acquired from a copy trade buy order. This endpoint allows the trader (signal provider) to close the corresponding spot position by specifying the original buy order ID.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `orderId` | int64 | Yes | The original copy trade buy order number to sell against |

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `orderId` | string | The sell order ID generated by the system |
| `clientOrderId` | string | Client-defined order ID (if provided) |
| `symbol` | string | Trading pair, e.g. `BTC-USDT` |
| `side` | string | Order direction — `SELL` |
| `type` | string | Order type, e.g. `MARKET` |
| `origQty` | string | Original order quantity |
| `executedQty` | string | Filled quantity |
| `avgPrice` | string | Average fill price |
| `status` | string | Order status: `NEW`, `PARTIALLY_FILLED`, `FILLED`, `CANCELED` |
| `createTime` | int64 | Order creation timestamp in milliseconds |
| `updateTime` | int64 | Last update timestamp in milliseconds |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "orderId": "1736011869418901234",
    "symbol": "BTC-USDT",
    "side": "SELL",
    "type": "MARKET",
    "origQty": "0.001",
    "executedQty": "0.001",
    "avgPrice": "65000.00",
    "status": "FILLED",
    "createTime": 1704067200000,
    "updateTime": 1704067201000
  }
}
```

## 2. Personal Trading Overview

### Personal Trading Overview

`GET /openApi/copyTrading/v1/spot/traderDetail`

Query the current trader's personal copy trade spot performance overview, including total profit, win rate, and trade statistics.

**Parameters:**

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `totalProfit` | string | Total realized profit (USDT) across all closed spot copy trade positions |
| `totalOrders` | int64 | Total number of spot copy trade orders placed |
| `winOrders` | int64 | Number of profitable orders |
| `lossOrders` | int64 | Number of losing orders |
| `winRate` | string | Win rate as a decimal, e.g. `"0.65"` = 65% |
| `totalFollowers` | int64 | Number of followers currently copying this trader |
| `totalCopyAmount` | string | Total amount being copied by followers (USDT) |
| `avgProfitRate` | string | Average profit rate per trade |
| `createTime` | int64 | Trader account creation timestamp in milliseconds |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "totalProfit": "1234.56",
    "totalOrders": 120,
    "winOrders": 78,
    "lossOrders": 42,
    "winRate": "0.65",
    "totalFollowers": 35,
    "totalCopyAmount": "50000.00",
    "avgProfitRate": "0.032"
  }
}
```

## 3. Profit Summary

### Profit Summary

`GET /openApi/copyTrading/v1/spot/profitHistorySummarys`

Query aggregated profit summary for the copy trade spot account over a specified time range.

**Parameters:**

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `totalProfit` | string | Total realized profit (USDT) in the queried period |
| `totalLoss` | string | Total realized loss (USDT) in the queried period |
| `netProfit` | string | Net profit = totalProfit − totalLoss |
| `profitRate` | string | Net profit rate as a decimal |
| `totalOrders` | int64 | Total closed orders in the queried period |
| `winOrders` | int64 | Number of profitable orders in the queried period |
| `winRate` | string | Win rate as a decimal |
| `startTime` | int64 | Actual start of the queried period (milliseconds) |
| `endTime` | int64 | Actual end of the queried period (milliseconds) |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "totalProfit": "856.20",
    "totalLoss": "213.40",
    "netProfit": "642.80",
    "profitRate": "0.031",
    "totalOrders": 45,
    "winOrders": 30,
    "winRate": "0.667",
    "startTime": 1704067200000,
    "endTime": 1706745600000
  }
}
```

## 4. Profit Details

### Profit Details

`GET /openApi/copyTrading/v1/spot/profitDetail`

Query per-order profit details for copy trade spot orders, paginated. Returns a breakdown of profit/loss for each individual completed copy trade spot order.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `startTime` | int64 | No | Start timestamp in milliseconds |
| `endTime` | int64 | No | End timestamp in milliseconds |
| `pageIndex` | int64 | Yes | Page number, must be greater than 0 (default: 1) |
| `pageSize` | int64 | Yes | Records per page, maximum 100 (default: 20) |

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `total` | int64 | Total number of records |
| `list` | array | Array of profit detail objects (see below) |

**Profit Detail Object:**

| Field | Type | Description |
|-------|------|-------------|
| `orderId` | string | Buy order ID |
| `symbol` | string | Trading pair, e.g. `BTC-USDT` |
| `side` | string | Original order direction — `BUY` |
| `buyPrice` | string | Average buy price |
| `sellPrice` | string | Average sell price |
| `qty` | string | Order quantity |
| `profit` | string | Realized profit (USDT); negative means loss |
| `profitRate` | string | Profit rate as a decimal |
| `commission` | string | Trading commission charged (USDT) |
| `createTime` | int64 | Buy order creation timestamp (milliseconds) |
| `closeTime` | int64 | Sell order completion timestamp (milliseconds) |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "total": 45,
    "list": [
      {
        "orderId": "1736011869418901234",
        "symbol": "BTC-USDT",
        "side": "BUY",
        "buyPrice": "63500.00",
        "sellPrice": "65000.00",
        "qty": "0.001",
        "profit": "1.50",
        "profitRate": "0.0236",
        "commission": "0.065",
        "createTime": 1704067200000,
        "closeTime": 1704070800000
      }
    ]
  }
}
```

## 5. Query Historical Orders

### Query Historical Orders

`GET /openApi/copyTrading/v1/spot/historyOrder`

Query historical spot copy trade orders for the current account, paginated. Returns both open and closed orders placed as part of copy trading activity.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `symbol` | string | No | Filter by trading pair, e.g. `BTC-USDT` |
| `startTime` | int64 | No | Start timestamp in milliseconds |
| `endTime` | int64 | No | End timestamp in milliseconds |
| `pageIndex` | int64 | Yes | Page number, must be greater than 0 (default: 1) |
| `pageSize` | int64 | Yes | Records per page, maximum 100 (default: 20) |

**Response `data`:**

| Field | Type | Description |
|-------|------|-------------|
| `total` | int64 | Total number of historical orders |
| `list` | array | Array of order objects (see below) |

**Order Object:**

| Field | Type | Description |
|-------|------|-------------|
| `orderId` | string | Order ID |
| `symbol` | string | Trading pair, e.g. `BTC-USDT` |
| `side` | string | Order direction: `BUY` or `SELL` |
| `type` | string | Order type: `MARKET` or `LIMIT` |
| `origQty` | string | Original order quantity |
| `executedQty` | string | Filled quantity |
| `price` | string | Order price (for LIMIT orders) |
| `avgPrice` | string | Average fill price |
| `cumQuote` | string | Total transaction value (USDT) |
| `status` | string | Order status: `NEW`, `PARTIALLY_FILLED`, `FILLED`, `CANCELED` |
| `profit` | string | Realized profit for this order (USDT) |
| `commission` | string | Commission charged (USDT) |
| `createTime` | int64 | Order creation timestamp (milliseconds) |
| `updateTime` | int64 | Last update timestamp (milliseconds) |

**Example Response:**
```json
{
  "code": 0,
  "msg": "",
  "data": {
    "total": 120,
    "list": [
      {
        "orderId": "1736011869418901234",
        "symbol": "BTC-USDT",
        "side": "BUY",
        "type": "MARKET",
        "origQty": "0.001",
        "executedQty": "0.001",
        "price": "0",
        "avgPrice": "63500.00",
        "cumQuote": "63.50",
        "status": "FILLED",
        "profit": "0",
        "commission": "-0.0635",
        "createTime": 1704067200000,
        "updateTime": 1704067201000
      }
    ]
  }
}
```

## Error Codes

Standard BingX error codes apply. Common errors for Copy Trade endpoints:

| Code | Description |
|------|-------------|
| `0` | Success |
| `-1100` | Illegal characters in a parameter |
| `-1102` | Mandatory parameter was not sent or is empty |
| `-1121` | Invalid symbol |
| `-2013` | Order does not exist |
| `-2014` | API key format invalid |
| `-2015` | Invalid API key, IP, or permissions |
| `80001` | Copy trade order not found |
| `80002` | Order already closed or not eligible for sell |
