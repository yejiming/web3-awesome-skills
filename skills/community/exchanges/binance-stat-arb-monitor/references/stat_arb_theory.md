# 统计套利策略理论

## 什么是统计套利？

统计套利（Statistical Arbitrage，简称 Stat Arb）是一种基于数学和统计方法的量化交易策略。它的核心思想是：

1. **发现相关性**：找到两个或多个具有统计相关性的资产
2. **建立均值模型**：计算它们之间价格的长期均衡关系
3. **捕捉偏离**：当价格偏离均衡时，做多被低估的资产，做空被高估的资产
4. **回归收益**：期待价格回归均值，从中获利

## ETH/BTC 统计套利

### 策略逻辑

ETH 和 BTC 都是加密货币，它们之间存在一定的相关性。当 ETH/BTC 比率偏离历史均值时：

- **z-score < -3.0**：ETH 相对 BTC 便宜 → 做多 ETH + 做空 BTC
- **z-score > +3.0**：ETH 相对 BTC 贵 → 做空 ETH + 做多 BTC

### 为什么有效？

1. **均值回归**：历史数据表明，ETH/BTC 比率会围绕均值波动
2. **对冲风险**：多空对冲消除了市场系统性风险
3. **统计特性**：基于大数定律，偏离会回归

### 风险因素

1. **极端行情**：黑天鹅事件可能导致价差持续扩大
2. **手续费**：交易成本对收益影响显著
3. **滑点**：实际成交价可能与预期不同
4. **资金费率**：持仓成本可能侵蚀收益

## z-score 详解

### 计算公式

```
z = (X - μ) / σ
```

- X：当前值
- μ：历史均值
- σ：标准差

### 解读

| z-score | 含义 | 信号 |
|---------|------|------|
| 0 | 正好在均值 | 无 |
| ±1 | 偏离 1 个标准差 | 正常波动 |
| ±2 | 偏离 2 个标准偶 | 值得关注 |
| ±3 | 偏离 3 个标准差 | 强烈信号 |
| ±4+ | 极端偏离 | 非常罕见，机会/风险 |

### 阈值设置

- **开仓阈值**：通常 ±2.5 到 ±3.0
- **平仓阈值**：通常 ±0.5 到 ±1.0
- **止损阈值**：通常 ±4.0 到 ±5.0
